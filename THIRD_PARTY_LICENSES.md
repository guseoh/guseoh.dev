## Brand and trademark use

Brand icons are used only to identify the related technology in blog category
navigation. Their use does not imply endorsement, certification, sponsorship,
or an official affiliation with any brand owner.

- Brand icons are not used as this blog's logo or favicon.
- SVG geometry and aspect ratios from Simple Icons are not altered. Icons
  inherit the surrounding text color so the navigation remains accessible in
  light, dark, hover, focus, and active states.
- No elements are removed from or added to a brand icon.
- Open-source icon licenses and trademark permissions are separate. Brand
  guidance should be checked again before an icon is added or its use changes.

Relevant official guidance for the icons registered in this project includes:

- Spring: https://spring.io/trademarks
- Git: https://git-scm.com/downloads/logos
- Docker: https://www.docker.com/company/newsroom/media-resources/
- PostgreSQL: https://www.postgresql.org/about/policies/trademarks/

The MySQL and GitHub brand icons are intentionally not registered. Their
official guidance does not clearly permit the blog's required `currentColor`
treatment across all light, dark, hover, focus, and active states:

- MySQL's third-party logo page grants no-permission use for its Conditional
  Use Logos under specified terms: https://www.mysql.com/about/legal/logos.html
- GitHub limits its logo palette to white, black, and limited grey or green
  cases: https://github.com/logos

MySQL therefore uses the project-authored Database icon, while GitHub and
GitHub Actions use the project-authored generic Repository icon. These are
conceptual substitutes and are not altered versions of either brand logo.

The general Database, Network, OS, Board, and Repository icons are simple SVG
line icons authored within this project. No general-purpose third-party icon
library is used for them.
